# 商品文件下载功能（u.bu31.com 商城）

> 需求：商品可上传一个小文件（如安装包 / 资料），客户**购买并支付成功后**可在支付成功页直接下载。
> 设计决策（已与用户确认）：下载资源模式 · 仅已购客户可下载 · 单文件。

## 数据流
1. 后台编辑商品 →「商品下载文件」区块 → 选文件 → 走已有的 `POST /api/admin/upload`（base64 上传到 R2 桶 `bu31-shop-files`，记录 `media` 表）→ 返回 `key`。
2. 保存商品时把 `file_key / file_name / file_size / file_uploaded_at` 写入 `products` 表（由已有的「更新商品」接口承担，新增 4 个字段）。
3. 客户支付成功 → 跳转到 `success.html?order=...&token=<query_token>`。
4. `success.js` 调 `GET /api/orders/:orderNo?token=...`（已返回 `file_key/file_name`），若订单已 PAID/DELIVERED 且商品有文件 → 显示「下载文件」按钮，链接 `/api/files/<query_token>`。
5. `GET /api/files/:token` 用订单 `query_token` 鉴权：订单必须已支付 → 取商品 `file_key` → 从 R2 流式返回文件（带 `Content-Disposition: attachment`）。未支付 / 无效 token → 403/404。

## 数据库变更
- `products` 表新增列：`file_key TEXT`、`file_name TEXT`、`file_size INTEGER`、`file_uploaded_at INTEGER`。
- 迁移脚本：`migrations/004_product_file.sql`（线上执行 `wrangler d1 execute bu31-shop-db --remote --file=./migrations/004_product_file.sql`）。
- `schema.sql` 已同步更新，新建库自带这 4 列。

## 涉及文件
- 后端 `src/worker.js`
  - `PUT /api/admin/products/:id` 写入 4 个文件字段
  - `GET /api/orders/:orderNo` 返回 `file_key/file_name`
  - `GET /api/files/:token` 新增公开下载端点（仅 PAID/DELIVERED 可下）
  - `DELETE /api/admin/products/:id/file` 新增后台删文件端点（清字段 + 删 R2 对象）
- 后台 `public/js/admin.js`
  - `openProductForm` 增加「商品下载文件」区块（显示当前文件 / 复制链接 / 移除 / 选择文件上传）
  - `uploadToR2` 改为返回完整对象 `{ok,key,url}`
  - 订单列表联系列增加「复制」按钮（`esc` 转义 + 剪贴板）
- 前台 `public/success.html` 增加 `#dlArea`；`public/js/success.js` 在已支付时渲染下载按钮

## 安全
- 下载端点不使用公开的 `/file/<key>` 代理，而是用订单 `query_token` 校验「已购买」状态，未支付无法下载。
- `query_token` 为 16 位随机 hex，不可猜测。
- 后台删除文件端点位于统一鉴权之后，仅管理员可调用。

## 限制
- 复用已有 base64 上传通道，建议单文件 < 30MB（`admin.js` 前端已做大小提示，实际受 Cloudflare 请求体上限约束）。
- 当前为单文件模式：一个商品一个下载文件；「移除」会清空字段并删除 R2 对象。
